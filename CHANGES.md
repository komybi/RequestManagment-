# Changes & New Files Summary

## Bug Fixes

### Fixed Internal Server Error
- **File**: `/lib/db.ts`
- **Issue**: MONGODB_URI was being checked at module load time, causing application crash
- **Fix**: Changed to check URI only when dbConnect() is called, allowing app to load without environment
- **Result**: App now loads successfully even without MongoDB connection initially

## Database Models - New & Updated

### New Models (5 Total)

1. **Request Model** (`/lib/models/Request.ts`)
   - Handles ID replacement and document requests
   - Fields: requestType, documentType, status, paymentFile, paymentVerified, deliveryMethod, etc.
   - Tracks unique tracking numbers for public access

2. **Notification Model** (`/lib/models/Notification.ts`)
   - Stores user notifications
   - Links to users and requests
   - Tracks read/unread status

3. **Letter Model** (`/lib/models/Letter.ts`)
   - Tracks formal letters generated by Registrar
   - Stores file URLs
   - Tracks if sent to Revenue staff

4. **Receipt Model** (`/lib/models/Receipt.ts`)
   - Tracks receipts generated by Revenue staff
   - Links to requests and generated-by user
   - Stores verification timestamp

### Updated Models

1. **User Model** (`/lib/models/User.ts`)
   - Added `studentId` field (optional)
   - Extended role enum: 'student' | 'admin' | 'registrar' | 'revenue'

## Known Limitations & TODO

1. Email sending requires Gmail app password setup
2. PDF generation happens in memory (can be optimized for large files)
3. QR codes are generated client-side (can be moved to server)
4. Payment verification is manual (can integrate payment gateway)
5. No audit logging yet
6. No rate limiting on API endpoints
7. No caching implemented
8. File uploads need cloud storage (AWS S3, etc.)

## Migration Guide for Existing Data

If migrating from old system:

1. Export existing requests from old database
2. Update status values to match new enum
3. Generate unique tracking numbers for each request
4. Create receipts/letters if needed
5. Migrate users and assign new roles
6. Update delivery methods to EMAIL or POSTAL
7. Verify all references are valid

## Performance Tips

1. Index MongoDB fields: email, status, studentId, role
2. Use connection pooling for MongoDB
3. Cache user sessions in Redis
4. Use CDN for static assets
5. Implement request pagination
6. Use async/await properly to prevent blocking
7. Monitor database query performance
